# Список вимог

## Functional Requirements (FR)

| ID   | Functional Requirement |
|------|-------------------------|
| **FR1**  | Система повинна дозволяти користувачеві реєструвати обліковий запис. |
| **FR2**  | Система повинна дозволяти авторизацію користувача. |
| **FR3**  | Користувач повинен мати можливість шукати товари. |
| **FR4**  | Користувач повинен мати можливість переглядати інформацію про товар. |
| **FR5**  | Користувач повинен мати можливість додавати товар до кошика. |
| **FR6**  | Система повинна дозволяти оформлення замовлення. |
| **FR7**  | Система повинна дозволяти перегляд статусу замовлення. |
| **FR8**  | Користувач може створити запит на повернення/обмін товару. |
| **FR9**  | Менеджер отримує нові замовлення. |
| **FR10** | Менеджер може підтвердити або відхилити замовлення. |
| **FR11** | Менеджер може змінювати статус замовлення. |
| **FR12** | Менеджер може обробляти повернення/обмін. |
| **FR13** | Менеджер може керувати товарами (додавати/редагувати). |
| **FR14** | Користувач може поставити питання менеджеру. |
| **FR15** | Менеджер може відповідати на питання користувача. |


## Non-Functional Requirements (NFR)

| ID   | Non-Functional Requirement |
|------|-----------------------------|
| **NFR1** | Час завантаження сторінки не повинен перевищувати 3 секунд. |
| **NFR2** | Система повинна підтримувати не менше 200 одночасних користувачів. |
| **NFR3** | Передача даних повинна відбуватися через HTTPS. |
| **NFR4** | Система повинна забезпечувати доступність 98%. |
| **NFR5** | Інтерфейс повинен бути адаптованим до мобільних пристроїв. |
| **NFR6** | Система повинна вести логування всіх ключових подій. |
| **NFR7** | Система має бути сумісна з основними браузерами. |
| **NFR8** | Архітектура повинна підтримувати горизонтальне масштабування. |
