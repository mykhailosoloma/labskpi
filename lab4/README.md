# Лабораторна робота №4: Моделювання бізнес-процесу (BPMN 2.0)

## 1. Загальна інформація
* **Назва сценарію:** Оплата товару (Payment Processing).
* **Тип діаграми:** BPMN 2.0 (Business Process Model and Notation).
* **Покриття вимог:** Деталізація алгоритму оплати, включаючи перевірку наявності товару (*Inventory Check*) та взаємодію із зовнішнім банком.

## 2. Учасники процесу (Pools & Lanes)
Діаграма розділена на зони відповідальності:

* **Покупець (Buyer):** Людина, яка вводить платіжні дані та приймає рішення про оплату.
* **Магазин (Web Store):** Основна система, що керує інтерфейсом та валідацією.
* **Склад (Inventory):** Внутрішній сервіс для перевірки залишків та резервування товару.
* **Платіжний шлюз (Bank):** Зовнішня система, яка виконує фінансову транзакцію.

## 3. Логіка виконання (Workflow)
Процес побудований на принципі **двоетапної фіксації** (Reserve -> Pay -> Commit):

1. **Валідація:** Користувач вводить дані, система перевіряє їх коректність.
2. **Перевірка складу:** Перед тим як брати гроші, Магазин запитує Склад. Якщо товар є — ставиться *тимчасовий резерв*.
3. **Оплата (Паралельність):**
   * Система показує анімацію очікування.
   * Одночасно відправляється захищений запит у Банк.
4. **Завершення:**
   * **Успіх:** Склад списує товар остаточно, Покупець отримує чек.
   * **Помилка:** Склад скасовує резерв (товар повертається у продаж), Покупець бачить причину відмови.
