## 1. Опис Проєкту та Мета

Метою цієї роботи є розробка логічної моделі даних для системи управління **Онлайн-магазином**. Модель побудована відповідно до принципів Entity-Relationship (ER) і відображає основні сутності та взаємозв'язки, необхідні для функціонування каталогу товарів, оформлення та обробки замовлень.

### Ключові Сутності (6 шт.):
1.  **Customer** (Клієнт)
2.  **Employee** (Співробітник)
3.  **Category** (Категорія)
4.  **Product** (Товар)
5.  **Order** (Замовлення)
6.  **OrderItem** (Деталь замовлення)

## 3. Опис Зв'язків

Усі зв'язки реалізовані за принципом **One-to-Many (1:N)**, що відображається зовнішніми ключами (FK) на стороні "N".

| Зв'язок (Relationship) | Сутності | Кратність | Реалізація |
| :--- | :--- | :--- | :--- |
| **Places** | Customer - Order | 1:N | `Order.customer_id` (FK, NOT NULL) |
| **Processes** | Employee - Order | 1:N | `Order.employee_id` (FK, **NULLABLE**) |
| **Contains (Category)**| Category - Product | 1:N | `Product.category_id` (FK, NOT NULL) |
| **Contains (Order)**| Order - OrderItem | 1:N | `OrderItem.order_id` (FK, NOT NULL) |
| **Is Item In** | Product - OrderItem | 1:N | `OrderItem.product_id` (FK, NOT NULL) |

### 4.2. Схема Даних (SQL DDL)

Цей код створює базу даних, реалізуючи логічну модель у реляційному форматі. Всі ключі та обмеження (PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE) суворо дотримуються ER-діаграми.

* **Файл:** `online_store_schema.sql`

| Таблиця | Первинний Ключ (PK) | Зовнішні Ключі (FK) | Обмеження |
| :--- | :--- | :--- | :--- |
| **Category** | `category_id` | - | - |
| **Product** | `product_id` | `category_id` | `price` DECIMAL(10, 2) |
| **Customer** | `customer_id` | - | `email` UNIQUE |
| **Employee** | `employee_id` | - | - |
| **Order** | `order_id` | `customer_id`, `employee_id` (NULLABLE) | `total_amount` DECIMAL(10, 2) |
| **OrderItem** | `order_item_id` | `order_id`, `product_id` | `unit_price` DECIMAL(10, 2) |
